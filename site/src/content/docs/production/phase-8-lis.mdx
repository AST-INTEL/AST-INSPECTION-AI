## What LIS Is

LIS (Laboratory/Inspection Information System) is CANAR's existing inspection reporting and tracking platform. It is the system of record for completed inspection jobs — client, asset, date, findings, recommendations, engineer sign-off.

Phase 8 connects the AI intelligence layer output to LIS as an input source. The AI generates structured findings. LIS receives them, presents them for engineer review, and manages the approval workflow it already handles today.

<Aside type="note" title="Neither system replaces the other">
The AI system does not replace LIS. LIS does not run the AI. They have a clean interface: AI generates → engineer reviews → LIS records. The boundary between them is a structured data handoff.
</Aside>

## Integration Scope (TBD)

The exact integration approach depends on the LIS API surface, which is reviewed after production approval. The options in order of preference:

| Option | Description | Complexity |
|---|---|---|
| **API integration** | AI system POSTs structured findings JSON to LIS API endpoint | Low — if LIS has a documented API |
| **Database integration** | AI writes directly to LIS database tables | Medium — requires schema access and DB permissions |
| **File-based handoff** | AI exports structured XML/CSV that LIS imports via existing batch import | Low — if LIS has an import function |
| **Manual bridge** | Engineer copy-pastes structured AI output into LIS | Zero dev work — worst-case fallback |
