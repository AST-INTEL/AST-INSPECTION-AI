# POC vs Production

## The Two-Track Strategy

This project is deliberately structured in two clean, independent tracks. The Proof of Concept track runs first, entirely on personal hardware, before any request is made to management. The Production track requires management approval to proceed.

**Management is never asked to approve something unproven. They are asked to approve something that already works.**

---

## Track 1: Proof of Concept

<Aside type="note" title="No approval needed — already underway">
The POC runs on a personal home machine. It uses CANAR's inspection data (with appropriate permissions) to validate the entire technical approach before any company resource commitment is requested.
</Aside>

| | Detail |
|---|---|
| **Machine** | Home workstation — Ryzen 7 7800X3D, RTX 4080 Super 16GB VRAM, 64GB RAM |
| **Status** | 🟢 Active — Phase 0 in progress |
| **Cost to company** | Zero |
| **Duration** | ~8 weeks from Phase 0 start |
| **Deliverable** | A working Docker container that processes real H30T imagery and produces an inspection report |

**What the POC proves:**
- The dual-LLM architecture works as designed
- YOLOv11 can be trained to recognise CANAR-specific defect types
- Reports generated by the system are comparable in quality to human-written reports
- The system is genuinely air-gapped — no outbound traffic during operation
- The Docker container can be transferred and deployed identically on different hardware

**What the POC does not prove:**
- Production-scale performance (the home machine runs models sequentially, not simultaneously)
- LIS integration (Phase 8 — production only)
- 32GB VRAM headroom from the RTX 5090

## Track 2: Production

<Aside type="caution" title="Management approval required before this track begins">
Production track begins only after: (1) the POC demo is complete, (2) management has reviewed and approved, and (3) IT has completed the VLAN isolation requirement.
</Aside>

| | Detail |
|---|---|
| **Machine** | CANAR work machine — Ryzen 9 Threadripper, RTX 5090 32GB VRAM, 256GB RAM |
| **Status** | 🟡 Pending approval |
| **Trigger** | Successful POC demo + management sign-off |
| **IT requirement** | VLAN isolation or dedicated isolated machine |
| **Phase added** | Phase 8 — LIS Integration |

**What changes from POC → Production:**

| Component | POC | Production |
|---|---|---|
| Inspection LLM | Qwen2.5 **14B** | Qwen2.5 **32B** |
| Operations LLM | Qwen2.5-Coder **7B** | Qwen2.5-Coder **14B** |
| LLM loading | Sequential (16GB VRAM) | Simultaneous (32GB VRAM) |
| Processing speed | ~X min/image | Faster — both models always warm |
| LIS integration | ❌ Not included | ✅ Phase 8 |
| Network isolation | Physical air-gap (home) | VLAN isolation (company network) |

## The Handover: POC → Production

The POC is specifically engineered for clean handover. The entire system is packaged as a single Docker container. Deploying on the work machine is:

<Steps>
1. Copy the Docker container file via USB
2. `docker load` the container on the work machine
3. Update 3 config values: `VRAM_LIMIT`, `MODEL_SIZE`, `ARCHIVE_PATH`
4. Run the smoke test suite
5. Production ready
</Steps>

No rebuild. No reinstall. No configuration drift between environments. The container that passed security verification on the POC machine is the same container running in production.

---

## What Management Is Being Asked to Approve

Specifically and only:

1. **IT department time** — VLAN isolation conversation and implementation (~1–2 days IT work)
2. **Use of the work machine** — dedicate the Threadripper/RTX 5090 workstation to this deployment
3. **Production model download** — one-time download of the 32B and Coder 14B model weights (approximately 25GB total) on company connection

That is the entire ask. No software licenses. No cloud subscriptions. No ongoing vendor costs. No new hardware purchases required.
