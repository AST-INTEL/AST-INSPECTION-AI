## Docker Container Export

```bash
# Finalise the container image
docker build -t ast-inspection-ai:v1.0-poc .

# Export to single transferable file
docker save ast-inspection-ai:v1.0-poc | gzip > ast-inspection-ai-v1.0-poc.tar.gz

# Verify the export
sha256sum ast-inspection-ai-v1.0-poc.tar.gz
# Record this hash — used to verify integrity on the production machine

# Test load on a clean environment (optional but recommended)
docker rmi ast-inspection-ai:v1.0-poc
docker load < ast-inspection-ai-v1.0-poc.tar.gz
# Run smoke test
```

## Production Transfer Runbook

Step-by-step instructions for deploying on the work machine. Written for a technically capable person who did not build the system.

<Steps>
1. **Prerequisites check** — Docker Desktop installed, NVIDIA drivers current, VLAN isolation confirmed by IT
2. **Copy the container** — `ast-inspection-ai-v1.0-poc.tar.gz` transferred via USB
3. **Verify integrity** — SHA-256 hash of the file must match the recorded value
4. **Load the container** — `docker load < ast-inspection-ai-v1.0-poc.tar.gz`
5. **Update the 3 config values** in `config/production.env`:
   - `VRAM_LIMIT=28` (28GB safe ceiling on RTX 5090's 32GB)
   - `MODEL_SIZE=32b` (triggers 32B model instead of 14B)
   - `ARCHIVE_PATH=/path/to/canar/inspection/archive`
6. **Run smoke test suite** — `docker exec ast-ai python tests/smoke_test.py`
7. **Verify GPU** — confirm both LLMs load simultaneously without exceeding VRAM ceiling
8. **Confirm network isolation** — netstat shows no external connections
9. **Production ready** — first real job can be submitted
</Steps>
