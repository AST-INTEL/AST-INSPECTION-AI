# Security Architecture

## Design Principle

Security is not configured after the system is built — it is the foundation the system is built on. Every architectural decision has a security dimension.

The security model has four layers, each independent. Defeating one layer does not defeat the others.

---

## Layer 1: File System Protection

| Control | Implementation | Effect |
|---|---|---|
| Archive read-only mount | Docker volume: `ro` flag | OS-level — no AI process can write to source data |
| Restricted pipeline user | Linux user with no delete permissions | Cannot be overridden by application code |
| Output folder structure | Separate `drafts/`, `pending_review/`, `approved/` directories | AI writes only to `drafts/` — no direct path to approved |
| Append-only logs | `chattr +a` on all log files | Logs cannot be deleted or truncated |
| Pre/post hash verification | SHA-256 of entire archive before and after every run | Any modification is mathematically detectable |

---

## Layer 2: Network Isolation

| Control | Implementation | Effect |
|---|---|---|
| Docker internal network | `network_mode: none` or internal-only network | Containers have no internet routing |
| Localhost binding | All services bind to `127.0.0.1` only | Streamlit UI only accessible from local machine |
| Firewall outbound block | Windows Firewall / UFW rules | Blocks all outbound from Docker bridge interface |
| Connection logging | All attempted connections logged | Any bypass attempt leaves a record |

**Verification:** Wireshark packet capture during the 30-minute Phase 6 demo session confirms zero packets to any external IP address.

---

## Layer 3: LLM Behavioural Controls

| Control | Implementation | Effect |
|---|---|---|
| Temperature = 0 | All inspection LLM calls use `temperature: 0` | Deterministic — no randomness in responses |
| Citation enforcement | System prompt requirement + output validation | LLM cannot make a claim without citing a source |
| Not-found fallback | Explicit prompt instruction | Returns "not found" rather than inventing information |
| Confidence gating | Detections below 0.6 confidence flagged | Low-confidence findings clearly marked for review |
| Output schema validation | Pydantic models validate all LLM JSON responses | Malformed or unexpected outputs rejected |
| Engineer review gate | All output goes to pending queue | Nothing reaches "approved" without human sign-off |

---

## Layer 4: Audit Trail

Every action in the system is logged. Logs are append-only and cannot be modified.

| Log | Contents |
|---|---|
| `pipeline.log` | Job submissions, stage completions, errors, processing times |
| `llm_calls.log` | Every LLM call — prompt hash, model, tokens used, response time |
| `vision.log` | Every YOLOv11 inference — image ID, detections, confidence scores |
| `security.log` | Permission violations, blocked network attempts, hash verification results |
| `health.log` | System health snapshots every 5 minutes — VRAM, queue depth, error count |
