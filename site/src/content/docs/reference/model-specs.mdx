## Upgrade Procedure

Upgrading models does not require rebuilding the Docker container from scratch.

```bash
# Pull new model weights inside the running container
docker exec ast-ai ollama pull qwen2.5:32b
docker exec ast-ai ollama pull qwen2.5-coder:14b

# Update config
# config/production.env: MODEL_SIZE=32b

# Run evaluation suite to confirm quality improvement
docker exec ast-ai python tests/eval_suite.py --compare 14b 32b

# If evaluation passes — tag new config as active
# If evaluation fails — roll back to previous MODEL_SIZE value
```

**Evaluation gate:** A new model is not deployed to production unless the evaluation suite shows measurably better performance on the CANAR benchmark set (defined during Phase 4). Rolling back is as simple as changing one config value.

---

## Continuous Improvement Model

| Mechanism | Frequency | What Happens |
|---|---|---|
| **Passive** | Every job | New job adds to ChromaDB — retrieval improves automatically |
| **Active** | Ongoing | Engineer corrections captured as fine-tuning dataset |
| **Scheduled** | Quarterly | Accumulated feedback used to fine-tune YOLOv11 and potentially LLM |

**Fine-tuning gate:** A fine-tuned model is only deployed if it scores higher on the benchmark set than the current production model. Version-controlled. Rollback always available.
